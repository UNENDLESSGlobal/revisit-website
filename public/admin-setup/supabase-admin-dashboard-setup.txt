Revisit admin dashboard Supabase setup

1. Create the admin access table:

create table if not exists public.admin_dashboard (
  email text primary key,
  created_at timestamptz not null default now()
);

2. Add the admin email rows that should be allowed into the dashboard:

insert into public.admin_dashboard (email)
values ('admin@example.com')
on conflict (email) do nothing;

3. Enable row level security:

alter table public.admin_dashboard enable row level security;
alter table public.profiles enable row level security;

4. Create a helper function for admin checks:

create or replace function public.is_dashboard_admin()
returns boolean
language sql
stable
as $$
  select exists (
    select 1
    from public.admin_dashboard
    where lower(email) = lower(coalesce(auth.jwt() ->> 'email', ''))
  );
$$;

grant execute on function public.is_dashboard_admin() to authenticated;

5. Policies for admin table self-check:

create policy "admin dashboard self read"
on public.admin_dashboard
for select
to authenticated
using (lower(email) = lower(coalesce(auth.jwt() ->> 'email', '')));

6. Policies for profiles table access from the admin dashboard:

create policy "dashboard admins can read profiles"
on public.profiles
for select
to authenticated
using (public.is_dashboard_admin());

create policy "dashboard admins can update profiles"
on public.profiles
for update
to authenticated
using (public.is_dashboard_admin())
with check (public.is_dashboard_admin());

7. Notes:
- The website signs users in with Supabase Auth, then checks whether the authenticated email exists in public.admin_dashboard.
- Keep using public environment variables in the website. Either prefix works:
  VITE_SUPABASE_URL or NEXT_PUBLIC_SUPABASE_URL
  VITE_SUPABASE_ANON_KEY or NEXT_PUBLIC_SUPABASE_ANON_KEY
  VITE_GOOGLE_APPS_SCRIPT_URL or NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL
- The dashboard updates public.profiles.plan and public.profiles.plan_expires_at after a payment is written to Google Sheets.
